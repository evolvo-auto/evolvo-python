# Project Overview

## Purpose
This repository contains the application source code under `app/` and lightweight process tracking for autonomous improvement work.

## Repository Structure
- `app/` — main implementation code.
- `tasks/` — queued task definitions (one task per markdown file).
- `completed_tasks/` — finished task files moved from `tasks/` with reflections.
- `README.md` — top-level project information.

## Contribution Flow
1. Pick a task from `tasks/`.
2. Implement the change in the codebase or docs.
3. Validate the change through basic verification commands.
4. Add a reflection section to the task file.
5. Move the completed task file into `completed_tasks/`.

## Task Tracking Convention
- Keep at least a small backlog in `tasks/`.
- Each task should include objective, steps, and completion criteria.
- Completed items must be preserved in `completed_tasks/` for history.
