# Contributing

Thanks for contributing to this repository.

## Setup

1. Create a Python virtual environment.
2. Install project dependencies (if/when declared).
3. Run project checks before opening a PR.

## Workflow

1. Add or refine tasks in `./tasks/`.
2. Work on one task at a time with concrete filesystem changes.
3. Verify your changes with shell commands.
4. Keep commits focused and descriptive.

## Coding Standards

- Prefer small, readable functions.
- Keep names explicit and consistent.
- Avoid unrelated refactors in task-scoped changes.
- Update docs when behavior or workflow changes.

## Pull Request Checklist

- [ ] Changes are scoped to a clear task.
- [ ] Files added/updated are verified locally.
- [ ] Documentation is updated when needed.
- [ ] No unrelated files were modified.
